了解获取 Windows 10 副本的几种方式

概述

技术术语

对于 Windows 10 副本，你需了解以下几个技术术语：

  • Windows 生命周期：每个 Windows 产品都有生命周期。生命周期从产品发布时算起，直至完全失去微软官方提供的安全更新为止。了解生命周期的关键日期可帮助你在对软件进行更新、升级或其他更改时，做出明智的决策。你可以在 Microsoft 官网 了解更多有关 Windows 生命周期的信息
  • ISO：光盘映像文件。在实际应用中，ISO 文件可直接通过 Windows 资源管理器或提供压缩解压缩服务的应用程序打开。ISO 文件还可用于制作 U 盘/光盘形式的 Windows 启动盘
  • Windows 激活：Windows 需要正版密钥才可以有效激活使用。对于 Windows 而言，只有使用从 Microsoft 或 Microsoft 授权经销商处获取的正版密钥来激活 Windows 才算是真正法律意义上的激活。具体内容可以参考我的个人理解
  • Windows 10 版本：广义上，Windows 10 版本分为 企业版/教育版/专业版/专业教育版/专业工作站版/家庭版 和 企业 LTSC/LTSB 版本。除企业 LTSC/LTSB 版本外，可统称为「Windows 10 半年频道」。针对 Windows 10 的功能更新每年将通过半年频道发布两次，定于每年的 3 月和 9 月。特殊情况下可能会推迟发布，但总体保持每年两次功能更新的节奏

声明

所有副本均为试用版，如有正版密钥可以有效激活，在此不提供任何激活和相关服务商业等用途的 Windows 10，请从 Microsoft 或 Microsoft 授权提供商处获取正版 Windows 10。本站不对非法使用 Windows 10 的行为负责

Windows 媒体创建工具

此工具可用于下载 Windows 10 版本的系统 ISO 副本文件，同时可在工具内选择直接封装于 U 盘等可移动设备以用作启动工具

Windows 媒体创建工具支持 32 位和 64 位不同版本的 Windows 10 ISO 内容创建

你可以在这个 页面 获取到最新版本的 Windows 媒体创建工具。你需要使用 Windows 设备访问这个页面

ISO 文件

ISO 文件是可以直接打开进行安装的一类文件，又名「光盘映像文件」。通过 Windows 媒体创建工具所获取到的 Windows 副本可以直接在媒体创建工具中转换为 ISO 文件存储

另外，你可以在这个 页面 获取到 ISO 文件。不限设备访问

MSDN 订阅

除了使用 Microsoft 官方提供的媒体创建工具获取镜像副本外，你还可以通过 MSDN 订阅或通过 MSDN,I tell you 获取 MSDN 版本的镜像副本。两者所获取到的镜像副本内容完全相同，且相较于媒体创建工具更加灵活。通过 MSDN 订阅获取到的文件均为 ISO 文件，开箱即用

  1. 打开 MSND，I tell you；
  2. 在左侧的类别选项中选择「操作系统」；
  3. 向下滚动步进器，然后选择你需要的 Windows 10 版本；
  4. 在右侧的详细信息栏中选择你需要的具体版本。「Consumer Editions」指消费者版本，「Business Editions」指商务版本。一般用户选择合适的体系架构的消费者版本即可；
  5. 点击对应项目下的「详细信息」按钮，复制下方提供 ed2k 链接至支持的下载工具进行下载即可

写在最后

正版和盗版是针对「授权」维度

「授权」的含义简单来说是「获得许可」，其表现形式通常是一段固定格式的非公开代码，俗称「激活码」（又被叫做「密钥（yuè）」）。由此正版可以理解为「已获得合法许可」，盗版则为「未获得合法许可」。至于是谁授权给谁，谁可以合法使用则是根据各个软件厂商对自己授权的定义。有的针对设备授权，有的针对个人或企业授权，并不是完全都相同，还有不同的版本区分

比如说厂商 A 将软件使用授权给企业 B，其授权范围为企业 B 下所有职工，那么员工 C 在入职企业 B 后即获得使用授权，在离职后即失去使用授权，在此期间任意使用都算是正版。如果授权的版本是「标准版」，而员工使用的是「高级版」，那么就属于盗版。如果电脑是员工 C 自行购买，入职时安装了该软件，离职时未删除，并且之后还继续使用该软件，那么也属于盗版

有一些企业早已经为自己员工购买了使用授权，而员工自己却以为使用的是盗版。这种情况也时有发生

厂商发行软件、发放激活码通常是两个独立行为。软件则一般具备对激活码的验证功能，验证通过则允许使用软件功能，不通过则阻止继续使用或自动关闭。但是软件只能做到验证激活码是否有效，不能验证其获得渠道是否合法，如果渠道非法，也是盗版

原版、纯净版、精简版、破解版、汉化版是针对「软件」维度

一般软件可以理解为「源码」、「编译」、「封装」、「发行」几个部分，其过程均为同一人/团队/单位/合作伙伴单独或共同完成

「原版」即为此过程经发行后的最终版本，每一个最终版本都有一个「文件指纹」，任何一个过程产生变化，最终文件指纹也会随之变化

在MSDN上，这个指纹就是 SHA1，是已知的作为验证原版的唯一手段

SHA1 只是一种指纹的算法，长度 40 位，常见的还有 MD5（长度 32 位）、SHA256（长度 64 位）、SHA384（长度 96 位）、SHA512（长度 128 位）等。MD5 是已知非安全算法，可以伪造出相同 MD5 值。但实际的文件不同，只能参考，不建议再作为唯一依据。SHA1 目前是发现有相同 SHA1 值的不同文件，但还不能指定一个 SHA1 值进行文件伪造，尚属安全

与原版相对应的版本我都归结为「修改版」，含义是对原版软件进行了二次修改，并已导致指纹与原版不同，文件名的修改不会影响指纹

因此也就至少存在三个方面无法验证。修改前的内容是什么？被修改的内容是什么？有没有一些没有说明出来但也修改了的内容？比如添加了恶意软件、木马、病毒？判断是否可信的依据仅为主观的对修改者的取信程度。根据个人 10 多年的软件使用经验，使用好评 10 万+，可能都是来自同一台电脑。通常很难验证的软件，就需要抱有怀疑态度。一般表面合理的修改只是为了掩饰潜在的目的

「纯净版」其中有一些人指的是原版，还有一些人指的是文件已经过修改，指纹也已经发生变化，但文件内所包含的内容均与原版一致。如果是指原版，可以按上述指纹验证，如果另一部分就很难验证

「精简版」已明确是属于修改版，通常针对软件中不经常使用的部分进行删除或屏蔽，意义在于节省软件运行过程中占用的资源

「破解版」已明确是属于修改版，通常仅针对软件中验证激活码的功能进行修改，意义在于不需要进行激活码验证，或者使用假激活码也可能验证通过，从而使用软件功能

「汉化版」含义是汉字化，中文化。其中有一部分是指软件是原版，但不是中文，只加入了中文对照。这种情况其实很少，常规操作是翻译人员向软件的官方提交中文对照，以语言包的形式提供。更多的另一部分通常是针对软件本身的操作界面进行修改，变更为中文使用更方便

综上，所有的修改，都需要对修改者的诚信进行判断

本人不对修改行为进行评价，毕竟有可信的，也有不可信的，是不是违反软件使用协议也不一定，开源软件还特别鼓励对软件进行修改。修改版也未必比原版差，原版的好处只是来源可信，有可验证的方法
